Preguntas Frecuentes

TOON (Token-Oriented Object Notation) es un formato de serialización de datos diseñado específicamente para prompts de LLM. Reduce el uso de tokens en un 30-60% en comparación con JSON mientras mantiene la legibilidad y estructura. El formato TOON utiliza un formato tabular para arrays y sintaxis TOON mínima para objetos, haciendo la conversión de JSON a TOON altamente eficiente.

TOON típicamente ahorra 30-60% en tokens LLM en comparación con JSON. Para conjuntos de datos grandes o respuestas de API, esto puede traducirse en ahorros de costos significativos al usar APIs como GPT-4 o Claude. El ahorro exacto depende de la estructura de tus datos.

Sí, TOON funciona con todos los LLMs principales incluyendo GPT-4, Claude, Gemini y LLaMA. Dado que TOON es un formato de texto, cualquier LLM puede leerlo y entenderlo. La mayoría de los LLMs modernos pueden analizar el formato TOON con instrucciones simples en tu prompt.

JSON vs TOON: TOON utiliza una sintaxis TOON más compacta diseñada para eficiencia de tokens. El formato de serialización TOON representa arrays como tablas con encabezados, usa puntuación mínima y elimina caracteres redundantes. JSON es más verboso con corchetes, comillas y comas. Cuando conviertes JSON a TOON para prompts LLM, la eficiencia de TOON reduce significativamente los costos.
