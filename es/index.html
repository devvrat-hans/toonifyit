El formato TOON utiliza un formato tabular para arrays y sintaxis TOON mínima para objetos, haciendo la conversión de JSON a TOON altamente eficiente.

JSON vs TOON: TOON utiliza una sintaxis TOON más compacta diseñada para eficiencia de tokens. El formato de serialización TOON representa arrays como tablas con encabezados, usa puntuación mínima y elimina caracteres redundantes. JSON es más verboso con corchetes, comillas y comas. TOON funciona con todos los principales proveedores de LLM incluyendo OpenAI (GPT-4, GPT-3.5), Anthropic (Claude), Google (Gemini), Meta (LLaMA) y otros. Como TOON es basado en texto, cualquier LLM puede entenderlo con instrucciones de prompting adecuadas.

TOON reduce tokens a través de tres optimizaciones: 1) Declarar nombres de campos una vez en encabezados de arrays en lugar de repetirlos, 2) Usar puntuación mínima, 3) Emplear estructura basada en indentación. Estos se combinan para lograr una reducción de tokens del 30-60% en comparación con JSON.
