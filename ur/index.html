اکثر پوچھے جانے والے سوالات

TOON (Token-Oriented Object Notation) ایک ڈیٹا سیریلائزیشن فارمیٹ ہے جو خاص طور پر ایل ایل ایم پرامپٹ کے لیے ڈیزائن کیا گیا ہے۔ یہ JSON کے مقابلے میں ٹوکن کی کھپت کو 30-60% کم کرتا ہے جبکہ قابل فہمی اور ڈھانچہ برقرار رکھتا ہے۔ TOON فارمیٹ arrays کے لیے tabular فارمیٹ اور objects کے لیے کم سے کم TOON نحو استعمال کرتا ہے، جو JSON سے TOON تبدیلی کو انتہائی موثر بناتا ہے۔

TOON عام طور پر JSON کے مقابلے میں ایل ایل ایم ٹوکنز پر 30-60% بچت کرتا ہے۔ بڑے ڈیٹاسیٹس یا API responses کے لیے، یہ GPT-4 یا Claude جیسے APIs استعمال کرتے وقت نمایاں لاگت کی بچت میں ترجمہ ہو سکتا ہے۔ حقیقی بچت آپ کے ڈیٹا کے ڈھانچے پر منحصر ہے۔

جی ہاں، TOON GPT-4، Claude، Gemini، اور LLaMA سمیت تمام بڑے LLMs کے ساتھ کام کرتا ہے۔ چونکہ TOON ایک ٹیکسٹ فارمیٹ ہے، کوئی بھی LLM اسے پڑھ اور سمجھ سکتا ہے۔ زیادہ تر جدید LLMs آپ کے prompt میں سادہ ہدایات کے ساتھ TOON فارمیٹ کو parse کر سکتے ہیں۔

بالکل۔ تمام تبدیلیاں JavaScript استعمال کرتے ہوئے آپ کے براؤزر میں مقامی طور پر ہوتی ہیں۔ آپ کا ڈیٹا کبھی بھی آپ کے ڈیوائس سے باہر نہیں جاتا یا کسی سرور کو نہیں بھیجا جاتا۔ کنورٹر صفحہ لوڈ ہونے کے بعد مکمل طور پر offline کام کرتا ہے۔

JSON بمقابلہ TOON: TOON ٹوکن کارکردگی کے لیے ڈیزائن کردہ زیادہ compact نحو استعمال کرتا ہے۔ TOON سیریلائزیشن فارمیٹ arrays کو headers کے ساتھ tables کے طور پر پیش کرتا ہے، کم سے کم punctuation استعمال کرتا ہے، اور فالتو characters کو ختم کرتا ہے۔ JSON brackets، quotes، اور commas کے ساتھ زیادہ verbose ہے۔ جب آپ ایل ایل ایم پرامپٹ کے لیے JSON کو TOON میں تبدیل کرتے ہیں، TOON کی کارکردگی نمایاں طور پر لاگت کو کم کرتی ہے۔
