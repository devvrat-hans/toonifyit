FAQ: Token Cost Reduction

Q1: Does TOON save money on output tokens?

A: No. Output tokens are generated by the LLM and can't be compressed by format. TOON only reduces input tokens (where most savings happen).

  • Input token reduction: 30-60%
  • Output token savings: 0%
  • Overall savings: 15-40% (depending on input/output ratio)

Q2: Are there any additional costs to implement TOON?

A: No hidden costs:

  • Free libraries available (open source)
  • No licensing fees
  • One-time implementation time (2-4 hours typical)
  • No infrastructure changes required

Q4: Will TOON work with my current stack?

A: Yes. TOON is format-agnostic:

  • Works with any programming language
  • Works with any framework (LangChain, LlamaIndex, etc.)
  • Zero infrastructure changes

Q5: What if my data doesn't compress well with TOON?
