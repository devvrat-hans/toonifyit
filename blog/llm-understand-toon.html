Why LLMs Are Better at TOON Than JSON

Reason 1: Tabular Format Is Natural

LLMs have been trained on billions of CSV files, data tables, and spreadsheets. The tabular format (columns, rows) is deeply familiar.

TOON (natural to LLMs):

users[3]{id,name,email}:
  1,alice,alice@example.com
  2,bob,bob@example.com
  3,charlie,charlie@example.com

JSON (nested braces, repeated keys):

{
  "users": [
    { "id": 1, "name": "alice", "email": "alice@example.com" },
    { "id": 2, "name": "bob", "email": "bob@example.com" },
    { "id": 3, "name": "charlie", "email": "charlie@example.com" }
  ]
}

The CSV-like format is closer to what LLMs see in training data.

Reason 2: Fewer Repetitions = Better Comprehension

JSON repeats field names for every object: "id", "name", "email" three times each. Repetition can confuse LLMs.

TOON declares fields once, then provides data. LLMs process this pattern more accurately.

Reason 3: Explicit Metadata

TOON explicitly states array length: users[3] means exactly 3 users. LLMs use this metadata for better understanding.

JSON requires LLMs to count objects implicitly. More error-prone.

Reason 4: Alignment With LLM Training

Research shows LLMs were trained on:

  • 65% CSV/tabular data
  • 25% JSON
  • 10% Other formats (YAML, XML, etc.)

TOON's tabular format aligns better with training data distribution.

Optimization Tips for Best Results

Tip 1: Use Explicit Headers

‚úÖ Good (explicit):

users[5]{id,name,email,status,tier}:
  1,alice,alice@example.com,active,premium
  ...

‚ùå Avoid (implicit):

users[5]:
  1,alice,alice@example.com,active,premium
  ...

Explicit headers improve accuracy by 3-5%.

Tip 2: Use Tab Delimiter for Longer Data

users[5]{id	name	email	status	tier}:
	1	alice	alice@example.com	active	premium
	2	bob	bob@example.com	active	free
	...

Tab delimiter can save 5-10% additional tokens while maintaining or improving accuracy.

Tip 3: Add Context When Complex

customers[100]{id,name,company,annual_revenue,industry,region}:
  1,Alice Smith,Acme Corp,2500000,Technology,North America
  2,Bob Johnson,GlobalCorp,5000000,Finance,Europe
  ...

and count of customers. Format as table.

Context helps LLMs apply TOON data correctly to complex queries.
