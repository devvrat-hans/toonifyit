What is Tabular Array in TOON Format?

Complete Technical Guide with Detection Algorithm and Code Examples

The tabular array is the core concept that makes TOON format revolutionary. Understanding how tabular arrays work is essential for developers wanting to optimize data structures and achieve massive token savings when you convert JSON to TOON.

This guide explains tabular arrays from fundamentals through advanced implementation‚Äîwith real code examples and practical applications.

üéØ Quick Definition
A tabular array is a uniform collection of objects where every object has identical keys in the same order, with only primitive values. TOON compresses these into CSV-like format, achieving 50-60% token reduction.

What is a Tabular Array? The Complete Definition

A data structure qualifies as a tabular array if and only if ALL three conditions are met:

1. Array of Objects (Not Primitives)

‚úÖ Tabular array candidate:

const users = [
  { id: 1, name: "Alice" },
  { id: 2, name: "Bob" }
];

‚ùå Not a tabular array (primitives):

const ids = [1, 2, 3];
const names = ["Alice", "Bob"];

2. Every Object Has Identical Keys in Same Order

‚úÖ Tabular array (all have {id, name, email}):

const users = [
  { id: 1, name: "Alice", email: "alice@example.com" },
  { id: 2, name: "Bob", email: "bob@example.com" },
  { id: 3, name: "Charlie", email: "charlie@example.com" }
];

‚ùå NOT tabular (missing email in second object):

const mixed = [
  { id: 1, name: "Alice", email: "alice@example.com" },
  { id: 2, name: "Bob" },  // ‚Üê missing 'email' field
  { id: 3, name: "Charlie", email: "charlie@example.com" }
];

‚ùå NOT tabular (different key order):

const wrongOrder = [
  { id: 1, name: "Alice", email: "alice@example.com" },
  { email: "bob@example.com", id: 2, name: "Bob" },  // ‚Üê different order
  { id: 3, name: "Charlie", email: "charlie@example.com" }
];

‚ùå NOT tabular (extra field in one object):

const extraField = [
  { id: 1, name: "Alice", email: "alice@example.com" },
  { id: 2, name: "Bob", email: "bob@example.com", phone: "555-1234" },  // ‚Üê extra field
  { id: 3, name: "Charlie", email: "charlie@example.com" }
];

3. All Values Must Be Primitives (No Objects/Arrays)

‚úÖ Tabular array (all primitive values):

const products = [
  { id: 1, name: "Laptop", price: 999.99, in_stock: true },
  { id: 2, name: "Mouse", price: 29.99, in_stock: false },
  { id: 3, name: "Keyboard", price: 79.99, in_stock: true }
];

‚ùå NOT tabular (nested object):

const withNested = [
  { id: 1, name: "Laptop", price: 999.99, specs: { cpu: "i7", ram: "16GB" } },  // ‚Üê nested object
  { id: 2, name: "Mouse", price: 29.99, specs: { dpi: 3200 } }
];

‚ùå NOT tabular (nested array):

const withArray = [
  { id: 1, name: "Alice", tags: ["admin", "developer"] },  // ‚Üê nested array
  { id: 2, name: "Bob", tags: ["user"] }
];
üí° Key Insight
All three requirements must be satisfied simultaneously. If any condition fails, the array is NOT tabular and TOON will use a different encoding format.

Tabular Array Encoding: How TOON Compresses It

Step 1: Detect Tabular Array

The TOON encoder runs a detection algorithm:

function isTabularArray(arr) {
  // Check 1: Must be array
  if (!Array.isArray(arr) || arr.length === 0) return false;
  
  // Check 2: First element must be object (not primitive, not array)
  const first = arr[0];
  if (typeof first !== 'object' || Array.isArray(first)) return false;
  
  // Check 3: All objects must have identical keys
  const firstKeys = Object.keys(first).sort();
  
  for (const item of arr) {
    const itemKeys = Object.keys(item).sort();
    if (JSON.stringify(firstKeys) !== JSON.stringify(itemKeys)) {
      return false;  // Different keys
    }
    
    // Check 4: All values must be primitives
    for (const value of Object.values(item)) {
      if (typeof value === 'object' && value !== null) {
        return false;  // Nested structure found
      }
    }
  }
  
  return true;  // ‚úÖ Is a tabular array!
}

Why Tabular Arrays Save So Many Tokens

The Repetition Problem in JSON

Every object in JSON repeats the field names:

{
  "customers": [
    { "id": 1, "name": "...", "revenue": 2500000, "industry": "..." },
    { "id": 2, "name": "...", "revenue": 5000000, "industry": "..." },
    { "id": 3, "name": "...", "revenue": 850000, "industry": "..." },
    ...
  ]
}

Field names appear 5 times: "id", "name", "revenue", "industry" repeated in each object = 20 tokens wasted on repetition.

The TOON Solution

Declare fields once:

customers[5]{id,name,revenue,industry}:
  1,Acme Corp,2500000,Technology
  2,Global Ent,5000000,Finance
  3,Local Store,850000,Retail
  ...

Field names appear once, then just the data. No repetition.

Implementation: How to Check if Array is Tabular

JavaScript Implementation

function isTabularArray(arr) {
  // Must be non-empty array
  if (!Array.isArray(arr) || arr.length === 0) return false;
  
  // First element must be object
  const first = arr[0];
  if (typeof first !== 'object' || Array.isArray(first) || first === null) {
    return false;
  }
  
  // Get first object's keys
  const keys = Object.keys(first);
  if (keys.length === 0) return false;
  
  // Check all elements
  for (const item of arr) {
    // Must be object
    if (typeof item !== 'object' || Array.isArray(item) || item === null) {
      return false;
    }
    
    // Must have same keys in same order
    const itemKeys = Object.keys(item);
    if (itemKeys.length !== keys.length) return false;
    
    for (let i = 0; i < keys.length; i++) {
      if (keys[i] !== itemKeys[i]) return false;
    }
    
    // All values must be primitives
    for (const value of itemKeys) {
      const v = item[value];
      if (v !== null && typeof v === 'object') return false;
    }
  }
  
  return true;
}

const users = [{id: 1, name: "Alice"}, {id: 2, name: "Bob"}];
console.log(isTabularArray(users));  // true

const mixed = [{id: 1}, {name: "Bob"}];
console.log(isTabularArray(mixed));  // false

Python Implementation

def is_tabular_array(arr):
    # Must be non-empty list
    if not isinstance(arr, list) or len(arr) == 0:
        return False
    
    # First element must be dict
    first = arr[0]
    if not isinstance(first, dict):
        return False
    
    # Get first dict's keys
    keys = list(first.keys())
    if len(keys) == 0:
        return False
    
    # Check all elements
    for item in arr:
        # Must be dict
        if not isinstance(item, dict):
            return False
        
        # Must have same keys in same order
        item_keys = list(item.keys())
        if item_keys != keys:
            return False
        
        # All values must be primitives
        for v in item.values():
            if isinstance(v, (dict, list)):
                return False
    
    return True

# Usage
users = [{"id": 1, "name": "Alice"}, {"id": 2, "name": "Bob"}]
print(is_tabular_array(users))  # True

mixed = [{"id": 1}, {"name": "Bob"}]
print(is_tabular_array(mixed))  # False

Best Practices for Tabular Arrays

Practice 1: Ensure Uniform Structure

// ‚úÖ Good: Validate first

  useFallbackMethod(data);

Practice 2: Normalize Before Checking

// ‚ùå Bad: Keys in different order
const data = [
  { id: 1, name: "Alice", email: "a@example.com" },
  { email: "b@example.com", id: 2, name: "Bob" },
  { name: "Charlie", email: "c@example.com", id: 3 }
];
// isTabularArray(data) ‚Üí false

// ‚úÖ Good: Normalize first
function normalizeObjects(arr) {
  if (arr.length === 0) return arr;
  const keys = Object.keys(arr[0]).sort();
  
  return arr.map(obj => {
    const normalized = {};
    for (const key of keys) {
      normalized[key] = obj[key];
    }
    return normalized;
  });
}

const normalized = normalizeObjects(data);
// Now isTabularArray(normalized) ‚Üí true

Practice 3: Explicit Delimiter Choice

// ‚ùå Default comma (might conflict with data)
const csv = encode(data, { delimiter: ',' });

// ‚úÖ Good: Tab delimiter for data-heavy content
const toon = encode(data, { delimiter: '\t' });
// 5-10% additional token savings
