JSON to TOON Dönüştürücüsü Nasıl Kullanılır

1

JSON Verilerinizi Yapıştırın

JSON verilerinizi kopyalayıp giriş alanına yapıştırın. JSON to TOON dönüştürücümüz tüm geçerli JSON formatlarını destekler. Örnek bir JSON to TOON dönüştürmesini görmek için "Örnek Yükle"ye tıklayabilirsiniz.

2

TOON Format Seçeneklerini Özelleştirin

Tercih ettiğiniz ayırıcıyı (virgül, sekme veya dikey çizgi), girinti boyutunu seçin ve TOON sözdizimi çıktısını gereksinimlerinize göre özelleştirmek için uzunluk göstergelerini açıp kapatın.

3

TOON Format Çıktısı Alın

JSON'unuzun anında TOON formatına dönüştürüldüğünü izleyin. TOON çıktısı yazarken gerçek zamanlı olarak görüntülenip, verimli TOON serileştirme formatını gösterir.

4

TOON Verilerini Kopyalayın veya İndirin

TOON çıktısını panonuza kopyalamak için "Kopyala" düğmesini kullanın veya dönüştürülen JSON to TOON verilerinizi dosya olarak kaydetmek için "İndir" butonuna tıklayın.

Sık Sorulan Sorular

TOON (Token-Oriented Object Notation), özellikle LLM istekleri için tasarlanmış bir veri serileştirme formatıdır. JSON ile karşılaştırıldığında token kullanımını %30-60 oranında azaltırken okunabilirlik ve yapıyı korur. TOON formatı, diziler için tablo biçimi ve nesneler için minimal TOON sözdizimi kullanır, bu da JSON to TOON dönüştürmesini yüksek oranda verimli hale getirir.

TOON, JSON ile karşılaştırıldığında tipik olarak LLM tokenlerinde %30-60 tasarruf sağlar. Büyük veri kümeleri veya API yanıtları için, bu GPT-4 veya Claude gibi API'ler kullanırken önemli maliyet tasarrufu anlamına gelebilir. Kesin tasarruf miktarı veri yapınıza bağlıdır.

Evet, TOON GPT-4, Claude, Gemini ve LLaMA dahil olmak üzere tüm büyük LLM'lerle çalışır. TOON bir metin formatı olduğundan, herhangi bir LLM bunu okuyabilir ve anlayabilir. Çoğu modern LLM, isteminizde basit talimatlarla TOON formatını ayrıştırabiliyor.

Kesinlikle. Tüm dönüştürmeler JavaScript kullanılarak tarayıcınızda yerel olarak gerçekleşir. Verileriniz hiçbir zaman cihazınızı terk etmez veya herhangi bir sunucuya gönderilmez. Dönüştürücü sayfa yüklendikten sonra tamamen çevrimdışı olarak çalışır.

TOON ile JSON: TOON, token verimliliği için tasarlanmış daha kompakt bir sözdizimi kullanır. TOON serileştirme formatı, dizileri başlıklı tablolar olarak temsil eder, minimal noktalama kullanır ve gereksiz karakterleri ortadan kaldırır. JSON, parantez, tırnak ve virgüllerle daha ayrıntılıdır. JSON'u TOON'a dönüştürdüğünüzde LLM istekleri için, TOON'un verimliliği maliyetleri önemli ölçüde azaltır. Ayrıntılı TOON ile JSON karşılaştırmasını okuyun.

JSON'u TOON'a dönüştürmek kolaydır: 1) JSON verilerinizi yukarıdaki dönüştürücümüze yapıştırın, 2) "TOON'a Dönüştür"e tıklayın, 3) TOON çıktısını kopyalayın. Programlı dönüştürme için JavaScript veya Python kitaplıklarımızı kullanabilirsiniz. Örnekler ve kod parçacıkları içeren tam JSON to TOON dönüştürme kılavuzumuzu görmek.

Evet! TOON tamamen tersine çevrilebilir - TOON'u JSON'a veri kaybı olmaksızın dönüştürebilirsiniz. Çift yönlü dönüştürücümüz hem JSON to TOON hem de TOON to JSON dönüştürmesini destekler. TOON verilerinizi yapıştırıp orijinal JSON yapınızı geri almak için "JSON'a Dönüştür" seçeneğini seçin.

TOON, veritabanı sorgu sonuçları, nesne listesi içeren API yanıtları, analitik verileri, CSV tarzı veri kümeleri ve e-ticaret ürün katalogları gibi düzenli tablo biçimindeki verilerle en iyi şekilde çalışır. Birden fazla nesne genelinde tekrarlanan alanlar içeren veriler en yüksek token tasarrufu (%40-60) sağlar. Küçük veri kümeleri veya derin iç içe yapılar daha küçük iyileştirmeler görebilir.

Evet, TOON iç içe nesneleri ve dizileri destekler. İç içe nesneler girinti tabanlı yapıyı (YAML gibi) kullanırken, iç içe diziler yapılarını korur. Ancak TOON'un token verimliliği, düz, düzenli verilerle en yüksektir. Karmaşık iç içe yapılar yine de çalışır ancak dramatik bir token azalması görmeyebilir.

Evet, hem TOON dönüştürücü aracı hem de TOON formatının kendisi tamamen ücretsiz olarak kullanılabilir. Dönüştürme sayısında, dosya boyutu kısıtlaması veya premium özelliklerde hiçbir limit yoktur. TOON formatı spesifikasyonu, bunu uygulamak isteyen herkes için açıktır.

TOON, OpenAI (GPT-4, GPT-3.5), Anthropic (Claude), Google (Gemini), Meta (LLaMA) ve diğerleri dahil olmak üzere tüm büyük LLM sağlayıcılarıyla çalışır. TOON metin tabanlı bir format olduğundan, uygun istemi talimatlarıyla herhangi bir LLM bunu anlayabilir.

TOON, üç anahtar optimizasyon aracılığıyla tokenları azaltır: 1) Alan adlarını dizi başlıklarında bir kez bildirerek bunları her nesne için tekrarlamak yerine, 2) Minimal noktalama kullanarak ve gereksiz parantez/süslü parantezleri ortadan kaldırarak, 3) Açık sınırlayıcılar yerine girinti tabanlı yapı kullanarak. Bunlar birleşerek JSON ile karşılaştırıldığında %30-60 token azalması sağlar.

Kesinlikle! TOON, ChatGPT API (GPT-4, GPT-3.5-turbo) ile mükemmel şekilde çalışır. JSON verilerinizi isteninize dahil etmeden önce TOON'a dönüştürün. Sistem mesajınıza TOON formatının kısa bir açıklaması ekleyin ve model bunu anlayacak ve işleyecektir. Veri açısından yoğun istekler için API maliyetlerinizi önemli ölçüde azaltabilir.

TOON dosyaları tipik olarak .toon uzantısını kullanır (ör. data.toon). TOON bir metin formatı olduğundan, bunu .txt dosyaları olarak da kaydedebilirsiniz. Dönüştürücümüzden indirilirken varsayılan uzantı .toon'dur ve format türünü açıkça belirtir.

Entegrasyon kolaydır: 1) TOON kütüphanesini diliniz için kurun (JavaScript için npm install @toon-format/toon, Python için pip install toon-format), 2) Kodla/kodunu çöz işlevlerini içeri aktarın, 3) LLM'ye göndermeden önce verilerinizi TOON'a dönüştürün, 4) İsteğe bağlı olarak yanıtları JSON'a geri dönüştürün. Kod örnekleri ve API referansı için belgelerimizi kontrol edin.

Evet, TOON LLM akışlı yanıtlarıyla çalışır. İsteklerinizde giriş tokenlarını azaltmak için isteminizde giriş verisi için TOON kullanırsınız. LLM'nin yanıtı (akışlı olsun veya olmasın) herhangi bir formatta olabilir. TOON çıktısına ihtiyacınız varsa, isteminize format talimatlarını ekleyin.

Çevrimiçi dönüştürücümüz 10MB'a kadar dosyaları işleyebilir, bu da çoğu kullanım durumu için uygundur. Daha büyük veri kümeleri için, komut satırı aracını kullanmanız veya TOON kitaplığını doğrudan uygulamanıza entegre etmenizi öneriyoruz. Kütüphane hiçbir dosya boyutu sınırına sahip değildir ve çok büyük veri kümelerini verimli bir şekilde işleyebilir.
